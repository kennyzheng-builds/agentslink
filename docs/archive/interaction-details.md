# A2A 对话关系与交互细节

## 参与方
- 主人 A ↔ Agent a
- 主人 B ↔ Agent b

## 四种对话关系

```
        主人 A ◄──────► Agent a
          │                │
          │   (人类决策)    │
          │                │
        主人 B ◄──────► Agent b
                  
          ▲                ▲
          └──── 加密通道 ────┘
           (Agent 直接对话)
```

| 关系 | 方向 | 内容 |
|-----|------|------|
| **1. 主人-Agent** | A↔a, B↔b | 指令、状态查询、补充信息 |
| **2. Agent-Agent** | a↔b | 技术细节、方案尝试 |
| **3. 主人-主人** | A↔B | 建联确认、高层决策 |
| **4. 跨层传递** | a→b→B, b→a→A | 有价值信息同步 |

---

## 场景1：需要主人配合/确认

### 触发条件
- 需要外部授权（如登录飞书后台）
- 涉及敏感操作（修改配置、删除数据）
- 方案有多个选项需要决策
- 尝试失败需要人类介入

### 处理流程
```
Agent a 遇到需要主人 A 配合的情况
        ↓
a 暂停与 b 的对话
        ↓
a 通知 A："需要你登录飞书开启 XX 权限，是否现在操作？"
        ↓
A 回复："好的，我开好了" / "稍等" / "换种方案"
        ↓
a 恢复与 b 的对话，同步进展
```

### 消息格式
```
🤖 需要你的配合

当前进展：正在解决飞书权限问题
需要操作：登录飞书后台 → 应用管理 → 开启「通讯录读取」权限
预计时间：2分钟

[我已开启] [稍后处理] [换种方案]
```

## 场景2：主人想补充信息

### 方式1：主动打断
```
A 对 a 说："等等，我补充一下"
        ↓
a 暂停与 b 的对话
        ↓
A 补充信息
        ↓
a 将补充信息同步给 b
```

### 方式2：通过状态卡片介入
```
主窗口显示：
┌─────────────────────────────────────┐
│ 🤖 正在协作：飞书权限问题            │
│ 进度：诊断中 → 尝试方案A → [暂停]    │
│                                     │
│ [查看详情] [补充信息] [结束会话]     │
└─────────────────────────────────────┘

A 点击「补充信息」→ 输入补充内容 → a 同步给 b
```

### 方式3：自然语言随时插入
```
A："对了，那个错误码是 10003"
a："收到，我同步给对方"
（a 立即将补充信息发给 b）
```

## 场景3：有价值信息同步（A↔a 传递给 b）

### 什么是有价值信息
- 主人的补充说明
- 主人的决策偏好
- 新发现的线索
- 方案调整方向

### 同步机制
```
A 对 a 说："其实我不介意用临时方案"
        ↓
a 判断：这是决策偏好，对 b 有价值
        ↓
a 发给 b："我主人说可以接受临时方案"
        ↓
b 收到后，视情况同步给 B 或用于调整策略
```

### 信息分级
| 级别 | 内容 | 是否同步给对端主人 |
|-----|------|------------------|
| **关键决策** | "必须今天解决" / "预算不超过1000" | ✅ 必须同步 |
| **偏好倾向** | "优先简单方案" / "可以接受临时方案" | ⚠️ 视情况 |
| **背景信息** | "这个系统之前出过问题" | ✅ 有帮助就同步 |
| **闲聊/情绪** | "好烦啊这个" / "加油" | ❌ 不同步 |

## 消息流向图

```
主人 A 说："试试方案 B 吧"
        ↓
Agent a 收到
        ↓
├─→ 同步给 Agent b："我主人建议试试方案 B"
│       ↓
│   Agent b 收到，调整策略
│       ↓
│   b 视情况同步给主人 B："对方建议方案 B，你觉得呢？"
│
└─→ a 回复 A："好的，我正在和对方沟通方案 B"
```

---

## 关键设计原则

1. **主人随时可介入**：Agent 必须响应主人的任何指令
2. **Agent 判断价值**：不是所有主人-Agent 对话都同步给对端
3. **对端主人知情权**：关键决策必须让对端主人知道
4. **不打扰原则**：同步信息要简洁，不制造噪音
