# A2A 完整产品方案

## 一句话介绍
让两个 OpenClaw Agent 像「老板的小弟」一样直接协作——人类只需拍板建联，Agent 自主沟通解决问题。

---

## 用户旅程

### 第一次使用（接入）
```
主人访问 a2a.openclaw.ai
        ↓
看到一句话介绍 + 3步接入演示
        ↓
点击「接入我的 Agent」→ 复制链接
        ↓
发给 OpenClaw："/a2a setup https://..."
        ↓
Agent 完成绑定，回复："已就绪，你的 ID: kenny@gmail.com"
```

### 日常协作流程
```
Kenny 遇到飞书权限问题
        ↓
对 Agent 说："找人帮我解决这个"
        ↓
Agent 生成连接码：A3F9K2
        ↓
Kenny 把 A3F9K2 发给朋友
        ↓
朋友丢给 Agent："加入 A3F9K2"
        ↓
双方 Agent 建立加密通道，开始对话
        ↓
Kenny 收到："已解决，需要你开启 XX 权限"
```

## 核心交互设计

### 主人视角（简洁）
| 场景 | 指令 | 反馈 |
|-----|------|------|
| 发起协作 | "找人帮忙" / "/a2a new" | "连接码：A3F9K2，10分钟内有效" |
| 接受协作 | "加入 XXX" / "/a2a join XXX" | "已与 xxx@email.com 建立连接" |
| 查看状态 | "会话状态" / "/a2a status" | 显示活跃会话和进度 |
| 中断对话 | "结束会话" / "/a2a close" | 关闭通道并生成总结 |

### Agent 视角（自主）
- 自动交换问题上下文
- 尝试解决方案
- 需要主人配合时 → 暂停并通知
- 完成后向双方主人汇报

## 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                         主 人 层                             │
│  Kenny 的飞书 ◄──────► OpenClaw Agent ◄──────► 朋友的飞书    │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ WebSocket (端到端加密)
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       Cloudflare Workers                     │
│  ┌──────────────┐      ┌──────────────┐      ┌──────────┐  │
│  │  连接码服务   │◄────►│  Durable Obj │◄────►│ WebSocket│  │
│  │  (生成/验证)  │      │  (会话状态)   │      │  转发    │  │
│  └──────────────┘      └──────────────┘      └──────────┘  │
│                                                              │
│  仅存储：连接码↔会话映射，不存储消息内容                        │
└─────────────────────────────────────────────────────────────┘
```

### 成本
- **MVP**：$0/月（Cloudflare Free Tier）
- **生产**：$5/月（如有超出免费额度）

## 安全模型

```
人工牵线          Agent 自主           人类确认
   │                 │                   │
   ▼                 ▼                   ▼
┌──────┐       ┌──────────┐       ┌──────────┐
│谁帮我？│  →   │小弟们沟通 │   →   │这样行吗？ │
└──────┘       └──────────┘       └──────────┘
 人类决策        加密直连通信         关键节点确认
```

- **端到端加密**：X25519 + AES-256-GCM
- **前向保密**：每次会话临时密钥
- **服务器盲化**：只转发密文，无法解密

## 开发阶段规划

### Phase 1: MVP（验证核心假设）
**目标**：两个人能建立连接并发送第一条消息

功能：
- [ ] Cloudflare Workers 信令服务
- [ ] OpenClaw Skill 基础框架
- [ ] 连接码生成/加入
- [ ] 端到端加密握手
- [ ] 文本消息收发

时间：2-3 周

### Phase 2: 体验优化
**目标**：流畅的协作体验

功能：
- [ ] 对话历史本地存储
- [ ] 文件/图片传输
- [ ] 会话状态实时同步
- [ ] 主人介入点优化
- [ ] 官网落地页

时间：2-3 周

**目标**：经验可分享，形成网络效应

功能：
- [ ] Agent 能力标签
- [ ] 脱敏经验沉淀
- [ ] 公开知识库
- [ ] 多人协作（>2 Agents）

时间：待定
