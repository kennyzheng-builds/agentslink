# Agent-to-Agent 通信设计方案

## 核心原则
> **人在回路中（Human-in-the-loop）**：Agent 早期阶段，人类负责安全和匹配决策。

## 流程设计

### 1. 发起求助（人类操作）
```
我的 Agent → 我："遇到飞书权限问题，需要找懂飞书的 Agent"
我：选择联系朋友 A（因为我知道他有相关 Agent）
```

### 2. 牵线建立（人类操作）
```
我 → 朋友 A："我的 Agent 有个飞书问题，能让你的 Agent 帮忙看看吗？"
朋友 A：同意，触发 Agent 间建联
```

### 3. Agent 自主对话（无需人类传话）
```
我的 Agent ←→ 朋友 A 的 Agent
- 直接交换问题详情
- 尝试解决方案
- 必要时请求各自主人配合（如需要验证码、授权等）
```

### 4. 结果同步（Agent → 人类）
```
我的 Agent → 我："已解决，需要你在飞书后台开启 XX 权限"
朋友 A 的 Agent → 朋友 A："帮 Kenny 解决了飞书权限问题"
```

### 5. 关闭通信（人类确认）
```
我：确认问题解决，关闭会话
朋友 A：确认收到感谢，关闭会话
```

## 状态流转

```
[待牵线] → [牵线中] → [对话中] → [待确认] → [已关闭]
              ↓           ↓
           [被拒绝]   [需介入]
```

## 安全机制

| 阶段 | 人类控制点 |
|------|-----------|
| 建联前 | 决定是否牵线、选择找谁 |
| 对话中 | 可随时查看进度、中断对话 |
| 需操作时 | Agent 请求主人确认后才执行 |
| 结束时 | 双方主人确认后关闭通道 |

## MVP 功能清单

- [ ] 牵线请求发送/接收
- [ ] Agent 间消息通道
- [ ] 对话状态展示
- [ ] 结果同步给主人
- [ ] 手动关闭通道

## 后续可扩展

- Agent 能力标签（方便知道找谁）
- 历史经验沉淀
- 自动匹配推荐（在人工确认前提下）
